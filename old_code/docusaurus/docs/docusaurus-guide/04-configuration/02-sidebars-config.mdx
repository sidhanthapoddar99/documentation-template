# Sidebar Configuration (sidebars.js)

The `sidebars.js` file controls how your documentation sidebar is structured and organized. It determines the navigation hierarchy, grouping, and linking for your documentation sections.

## Sidebar Item Types

Docusaurus supports several types of sidebar items, each serving different organizational purposes:

### 1. Document References (String)

The simplest form - just reference a document by its ID:

```javascript
module.exports = {
  docs: [
    'intro',           // References docs/intro.md
    'getting-started', // References docs/getting-started.md
  ],
};
```

### 2. Categories

Group related documents together:

```javascript
{
  type: 'category',
  label: 'Tutorial',
  collapsed: false,          // Start expanded
  collapsible: true,         // Can be collapsed
  items: [
    'tutorial/basics',
    'tutorial/advanced',
  ],
}
```

#### Category Options:
- **`label`**: Display name in sidebar
- **`collapsed`**: Whether category starts collapsed
- **`collapsible`**: Whether users can collapse/expand
- **`className`**: Custom CSS class
- **`customProps`**: Custom properties for theming

### 3. Auto-generated Categories

Let Docusaurus automatically generate sidebar items from a directory:

```javascript
{
  type: 'autogenerated',
  dirName: 'guides',    // Auto-generate from docs/guides/
}
```

<Callout type="info" title="Autogenerated Behavior">
Autogenerated categories respect `_category_.json` files and automatically organize subdirectories as nested categories.
</Callout>

### 4. External Links

Add links to external resources:

```javascript
{
  type: 'link',
  label: 'GitHub Repository',
  href: 'https://github.com/myorg/myproject',
}
```

### 5. Document References (Object)

More control over document references:

```javascript
{
  type: 'ref',
  id: 'api/reference',
  label: 'API Reference',    // Custom label
}
```

### 6. HTML Items

Insert custom HTML into the sidebar:

```javascript
{
  type: 'html',
  value: '<hr>',
  defaultStyle: true,        // Apply default styling
}
```

## Multiple Sidebars

You can define multiple sidebars for different documentation sections:

```javascript
module.exports = {
  // Main documentation sidebar
  docs: [
    'intro',
    {
      type: 'category',
      label: 'Guides',
      items: ['guide1', 'guide2'],
    },
  ],
  
  // API documentation sidebar
  api: [
    'api/overview',
    {
      type: 'autogenerated',
      dirName: 'api/reference',
    },
  ],
  
  // Tutorial sidebar
  tutorial: [
    'tutorial/intro',
    'tutorial/basics',
    'tutorial/advanced',
  ],
};
```

## Sidebar Linking Strategies

### Generated Index Pages

Create category overview pages automatically:

```javascript
{
  type: 'category',
  label: 'Tutorial',
  link: {
    type: 'generated-index',
    title: 'Tutorial Overview',
    description: 'Learn the basics step by step',
    slug: '/tutorial',
    keywords: ['tutorial', 'basics'],
  },
  items: ['tutorial/step1', 'tutorial/step2'],
}
```

### Link to First Document

Make categories clickable by linking to their first document:

```javascript
{
  type: 'category',
  label: 'API Reference',
  link: {
    type: 'doc',
    id: 'api/overview',
  },
  items: ['api/overview', 'api/endpoints'],
}
```

### Custom Category Pages

Link to a custom page or external URL:

```javascript
{
  type: 'category',
  label: 'Examples',
  link: {
    type: 'generated-index',
    slug: '/examples',
  },
  items: [/* ... */],
}
```

## Advanced Sidebar Patterns

### Conditional Sidebar Items

Show different items based on environment or conditions:

```javascript
const isDev = process.env.NODE_ENV === 'development';

module.exports = {
  docs: [
    'intro',
    'getting-started',
    
    // Development-only items
    ...(isDev ? [
      {
        type: 'category',
        label: 'Development',
        items: ['dev/setup', 'dev/debugging'],
      },
    ] : []),
  ],
};
```

### Mixed Auto and Manual Organization

Combine autogenerated and manual sidebar organization:

```javascript
module.exports = {
  docs: [
    // Manual organization for key pages
    'intro',
    'quick-start',
    
    // Auto-generated for large sections
    {
      type: 'category',
      label: 'API Reference',
      link: {
        type: 'generated-index',
        title: 'API Documentation',
      },
      items: [
        {
          type: 'autogenerated',
          dirName: 'api',
        },
      ],
    },
    
    // Manual organization for specific order
    {
      type: 'category',
      label: 'Tutorials',
      items: [
        'tutorials/beginner',
        'tutorials/intermediate',
        'tutorials/advanced',
      ],
    },
  ],
};
```

## Sidebar Customization

## Sidebar Best Practices

<Tabs>
<TabItem value="organization" label="Organization">

### Logical Grouping
- Group related concepts together
- Use clear, descriptive category names
- Limit nesting to 2-3 levels

### Progressive Disclosure
- Start with essential information
- Use collapsed categories for advanced topics
- Put reference material in separate sections

</TabItem>
<TabItem value="navigation" label="Navigation">

### User Flow
- Order items by typical user journey
- Place getting-started content first
- End with reference and advanced topics

### Accessibility
- Use meaningful labels
- Maintain consistent naming conventions
- Avoid overly long category names

</TabItem>
<TabItem value="maintenance" label="Maintenance">

### Scalability
- Use autogenerated categories for large sections
- Keep manual organization for critical paths
- Regular review and reorganization

### Performance
- Avoid too many sidebar items (>50 per level)
- Use collapsed categories to reduce initial load
- Consider splitting large sidebars

</TabItem>
</Tabs>

## Common Sidebar Patterns

### Documentation Portal Pattern
```javascript
module.exports = {
  docs: [
    'overview',
    {
      type: 'category',
      label: 'Getting Started',
      collapsed: false,
      items: ['installation', 'quick-start', 'first-project'],
    },
    {
      type: 'category',
      label: 'Guides',
      items: [
        {type: 'autogenerated', dirName: 'guides'},
      ],
    },
    {
      type: 'category',
      label: 'API Reference',
      link: {type: 'generated-index'},
      items: [
        {type: 'autogenerated', dirName: 'api'},
      ],
    },
  ],
};
```

A well-organized sidebar is crucial for user experience. Users should be able to find information quickly and understand the logical flow of your documentation.

The sidebar configuration is powerful and flexible. Start simple with autogenerated sidebars, then add manual organization where needed for optimal user experience.
