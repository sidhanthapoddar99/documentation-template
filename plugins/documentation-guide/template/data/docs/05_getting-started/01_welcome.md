# Welcome

This is the starting point for **your** documentation. The framework copied this
file in when you ran `init`; replace it with whatever you want.

## What's already wired up

- A **Docs** section at `/docs` (this file)
- A **Blog** at `/blog` with a sample post
- An **Issues** tracker at `/issues`
- The framework's own **User Guide** at `/user-guide` — kept up-to-date by the
  framework, useful when you forget how something works

## Next steps

- Open `data/docs/05_getting-started/01_welcome.md` and edit this file
- Add another section: create `data/docs/10_<your-section>/` with a `settings.json`
- Customise the home page via `data/pages/home.yaml`
- Swap the placeholder logo in `assets/` for your own

The `XX_` prefix on filenames and folders controls ordering; URLs strip it
(`05_getting-started/01_welcome.md` → `/docs/getting-started/welcome`).
