# Frontmatter Reference

Frontmatter is the YAML metadata block at the top of every documentation file. It controls how pages appear in the sidebar, their SEO properties, and various display options.

## Basic Syntax

Frontmatter must be:
- At the **very beginning** of the file
- Enclosed between triple dashes `---`
- Valid YAML syntax

```markdown
---
title: Page Title
description: A brief description
sidebar_position: 1
---

# Page Content

Your markdown content starts here...
```

## Required Fields

### title

The page title displayed in the sidebar, browser tab, and as the main heading.

```yaml
title: Getting Started
```

- **Type:** `string`
- **Required:** Yes
- Used for: Sidebar label, `<title>` tag, `<h1>` if not overridden

## SEO & Metadata Fields

### description

A brief description for SEO and social sharing.

```yaml
description: Learn how to install and configure the documentation framework
```

- **Type:** `string`
- **Required:** No
- Used for: `<meta name="description">`, Open Graph description
- **Recommendation:** Keep between 150-160 characters

### keywords

An array of keywords for SEO.

```yaml
keywords:
  - documentation
  - getting started
  - installation
```

- **Type:** `string[]`
- **Required:** No
- Used for: `<meta name="keywords">`

### image

Path to an image for social sharing (Open Graph).

```yaml
image: /images/getting-started-cover.png
```

- **Type:** `string`
- **Required:** No
- Used for: `<meta property="og:image">`
- Should be an absolute path from the site root

## Sidebar Configuration

### sidebar_position

Numeric position in the sidebar. Lower numbers appear first.

```yaml
sidebar_position: 1
```

- **Type:** `number`
- **Required:** No
- **Default:** Alphabetical order by filename
- Pages with `sidebar_position` appear before those without

### sidebar_label

Custom label for the sidebar (different from title).

```yaml
title: Application Programming Interface Reference
sidebar_label: API Reference
```

- **Type:** `string`
- **Required:** No
- **Default:** Uses `title`
- Useful when the title is too long for the sidebar

## Content Organization

### tags

Categorize content with tags for filtering and organization.

```yaml
tags:
  - beginner
  - tutorial
  - react
```

- **Type:** `string[]`
- **Required:** No
- Can be used for filtering, tag pages, or related content

### category

Assign a primary category to the page.

```yaml
category: tutorials
```

- **Type:** `string`
- **Required:** No
- Used for grouping and filtering content

## Publishing Control

### draft

Mark a page as a draft to exclude it from production builds.

```yaml
draft: true
```

- **Type:** `boolean`
- **Required:** No
- **Default:** `false`
- Draft pages are visible in development but excluded from production

### published_at

Original publication date.

```yaml
published_at: 2024-01-15
```

- **Type:** `date`
- **Required:** No
- Format: `YYYY-MM-DD` or full ISO date

### updated_at

Last update date.

```yaml
updated_at: 2024-06-20
```

- **Type:** `date`
- **Required:** No
- Format: `YYYY-MM-DD` or full ISO date
- Useful for showing "Last updated" on pages

## Page Behavior

### hide_title

Hide the automatic title heading on the page.

```yaml
hide_title: true
```

- **Type:** `boolean`
- **Required:** No
- **Default:** `false`
- Use when you want a custom hero section or landing page

### hide_toc

Hide the table of contents (outline) on this page.

```yaml
hide_toc: true
```

- **Type:** `boolean`
- **Required:** No
- **Default:** `false`
- Useful for pages with minimal headings or custom layouts

### full_width

Render the page at full width without sidebar constraints.

```yaml
full_width: true
```

- **Type:** `boolean`
- **Required:** No
- **Default:** `false`
- Useful for landing pages or wide content like tables

## Complete Example

Here's a comprehensive example using multiple frontmatter options:

```yaml
---
title: Building Your First Component
description: A step-by-step guide to creating custom React components in the documentation framework
keywords:
  - react
  - components
  - tutorial
image: /images/components-tutorial.png

sidebar_position: 2
sidebar_label: First Component

tags:
  - beginner
  - react
  - tutorial
category: tutorials

draft: false
published_at: 2024-01-15
updated_at: 2024-06-20

hide_title: false
hide_toc: false
full_width: false
---

# Building Your First Component

In this tutorial, you'll learn...
```

## Minimal Example

For simple pages, you only need:

```yaml
---
title: Quick Start
---
```

## Frontmatter Validation

The framework validates frontmatter at build time. Invalid values will cause build errors:

```
Error: Invalid frontmatter in docs/my-page.mdx
  - "sidebar_position" expected number, received string
```

Common validation errors:
- Missing required `title` field
- Wrong type (string instead of number, etc.)
- Invalid date format
- Unrecognized field names (typos)

## Tips & Best Practices

1. **Always include `title`** - It's the only required field
2. **Add `description`** - Improves SEO and appears in search results
3. **Use `sidebar_position`** - For predictable ordering
4. **Keep `sidebar_label` short** - Sidebar space is limited
5. **Use `draft: true`** - For work-in-progress content
6. **Date format** - Always use `YYYY-MM-DD` for consistency
