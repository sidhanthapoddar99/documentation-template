# Using Assets

Assets are supporting files like code snippets, images, configuration examples, and diagrams that you reference in your documentation. This guide explains how to organize and use them effectively.

## The Assets Folder

Every documentation section can have an `assets` folder that is **automatically excluded** from the sidebar and routing.

```
components/
├── index.mdx
├── buttons.mdx
├── cards.mdx
└── assets/              # ← Not shown in sidebar
    ├── button-example.tsx
    ├── card-example.tsx
    └── screenshot.png
```

## Why Use Assets?

1. **Keep code separate** - Easier to maintain and syntax-highlight
2. **Avoid MDX escaping issues** - Raw files don't need escaping
3. **Reuse across pages** - Import the same asset in multiple files
4. **Clean organization** - Supporting files don't clutter the sidebar

## Importing Code Snippets

### Basic Import

Use the `raw-loader` to import code files as strings:

```mdx
import ButtonCode from '!!raw-loader!./assets/button-example.tsx';

<CodeBlock language="tsx">{ButtonCode}</CodeBlock>
```

### With Collapsible Code Block

```mdx
import ConfigExample from '!!raw-loader!./assets/config-example.json';

<CollapsibleCodeBlock title="Configuration Example" language="json">
  {ConfigExample}
</CollapsibleCodeBlock>
```

### Multiple Files

```mdx
import ComponentCode from '!!raw-loader!./assets/component.tsx';
import StylesCode from '!!raw-loader!./assets/styles.css';
import UsageCode from '!!raw-loader!./assets/usage.tsx';

## Component

<CodeBlock language="tsx">{ComponentCode}</CodeBlock>

## Styles

<CodeBlock language="css">{StylesCode}</CodeBlock>

## Usage

<CodeBlock language="tsx">{UsageCode}</CodeBlock>
```

## Supported File Types

You can store any file type in the assets folder:

### Code Files

| Extension | Language | Use Case |
|-----------|----------|----------|
| `.ts`, `.tsx` | TypeScript | React/TypeScript examples |
| `.js`, `.jsx` | JavaScript | JavaScript examples |
| `.py` | Python | Python code snippets |
| `.go` | Go | Go examples |
| `.rs` | Rust | Rust examples |
| `.css` | CSS | Style examples |
| `.json` | JSON | Configuration files |
| `.yaml`, `.yml` | YAML | Configuration files |
| `.sh`, `.bash` | Shell | Command examples |
| `.sql` | SQL | Database queries |

### Media Files

| Extension | Type | Use Case |
|-----------|------|----------|
| `.png`, `.jpg`, `.gif` | Images | Screenshots, diagrams |
| `.svg` | Vector | Icons, logos |
| `.mp4`, `.webm` | Video | Demo videos |

### Other Files

| Extension | Use Case |
|-----------|----------|
| `.mermaid` | Mermaid diagram source |
| `.dot` | Graphviz diagram source |
| `.md` | Markdown snippets |
| `.env.example` | Environment templates |

## Importing Images

### Standard Import

```mdx
import Screenshot from './assets/screenshot.png';

<img src={Screenshot} alt="Feature screenshot" />
```

### With Custom Styling

```mdx
import Diagram from './assets/architecture-diagram.png';

<figure>
  <img
    src={Diagram}
    alt="System architecture"
    className="rounded-lg border"
  />
  <figcaption>Figure 1: System Architecture</figcaption>
</figure>
```

## Organizing Large Sections

For sections with many assets, create sub-folders:

```
components/
├── index.mdx
├── buttons.mdx
├── cards.mdx
└── assets/
    ├── buttons/
    │   ├── primary.tsx
    │   ├── secondary.tsx
    │   └── icon-button.tsx
    ├── cards/
    │   ├── basic-card.tsx
    │   └── image-card.tsx
    └── shared/
        └── types.ts
```

Import with the full path:

```mdx
import PrimaryButton from '!!raw-loader!./assets/buttons/primary.tsx';
import SharedTypes from '!!raw-loader!./assets/shared/types.ts';
```

## Page-Specific Assets

For assets used by a single page, you can create a folder matching the page name:

```
configuration/
├── themes.mdx
├── layouts.mdx
└── assets/
    ├── themes/              # Assets for themes.mdx
    │   ├── dark-theme.css
    │   └── light-theme.css
    └── layouts/             # Assets for layouts.mdx
        ├── sidebar-layout.tsx
        └── full-width-layout.tsx
```

## Best Practices

### 1. Name Files Descriptively

```
# Good
assets/
├── button-primary-example.tsx
├── button-disabled-state.tsx
└── form-validation-schema.ts

# Avoid
assets/
├── code1.tsx
├── code2.tsx
└── example.ts
```

### 2. Keep Assets Close to Content

Place assets in the same section as the content that uses them:

```
components/
├── buttons.mdx        # References button assets
└── assets/
    └── button-*.tsx   # Button-related assets
```

### 3. Use Consistent Naming

```
# Pattern: [component]-[variant]-[purpose].[ext]
button-primary-example.tsx
button-secondary-disabled.tsx
card-image-with-actions.tsx
```

### 4. Document Complex Examples

Add comments in your asset files:

```tsx
// assets/form-example.tsx

/**
 * Example: Form with validation
 *
 * This example demonstrates:
 * - Field validation
 * - Error messages
 * - Submit handling
 */
export function FormExample() {
  // ...
}
```

### 5. Version Control Friendly

Keep assets small and focused. If an example file is getting too long, split it into multiple files.

## Common Patterns

### Full Component Example

```
components/tabs/
├── index.mdx
└── assets/
    ├── tabs-basic.tsx         # Basic usage
    ├── tabs-controlled.tsx    # Controlled state
    ├── tabs-with-icons.tsx    # With icons
    └── tabs-vertical.tsx      # Vertical layout
```

### API Documentation

```
api/endpoints/
├── users.mdx
└── assets/
    ├── users-list-request.sh      # curl example
    ├── users-list-response.json   # Response example
    ├── users-create-request.json  # Request body
    └── users-create-response.json # Response example
```

### Configuration Guide

```
configuration/
├── themes.mdx
└── assets/
    ├── theme-config.ts       # TypeScript config
    ├── theme-config.json     # JSON alternative
    └── custom-theme.css      # CSS variables
```

## Troubleshooting

### Import Not Found

```
Error: Can't resolve './assets/example.tsx'
```

- Check the file path is correct
- Ensure the file exists
- Verify the file extension

### Raw Loader Issues

```
Error: Module parse failed: Unexpected token
```

Make sure to use the `!!raw-loader!` prefix for code files:

```mdx
// ✓ Correct
import Code from '!!raw-loader!./assets/example.tsx';

// ✗ Wrong - will try to execute as JS
import Code from './assets/example.tsx';
```

### Image Not Displaying

- Check the file path
- Ensure the image format is supported
- Verify the import syntax matches the file type
