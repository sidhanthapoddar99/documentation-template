# Portal Overview

The Neuralock Management Portal is a modern web-based administration interface that provides comprehensive tools for managing the entire Neuralock ecosystem. Built with React, Vite, and shadcn/ui, it offers a responsive, intuitive interface for server management, role administration, and system monitoring.

## Authentication System

<MermaidDiagram content={AuthFlow} filename="auth-flow.mermaid" />

### Wallet-Based Authentication

The portal uses Web3 wallet authentication:

1. **Connect Wallet**: Support for MetaMask, WalletConnect, and other providers
2. **Sign Message**: Cryptographic proof of ownership
3. **Verify Role**: On-chain role verification
4. **Generate Session**: JWT token with role permissions
5. **Access Control**: Route and component-level authorization

<Callout type="info">
  <strong>Security Note:</strong> Sessions expire after 24 hours and require re-authentication. All sensitive operations require transaction signing.
</Callout>

### Technology Stack

- **Frontend Framework**: React 18 with TypeScript
- **Build Tool**: Vite for fast development and optimized builds
- **UI Components**: shadcn/ui for consistent, accessible components
- **State Management**: Redux Toolkit with RTK Query
- **Styling**: Tailwind CSS with custom theme system
- **Charts**: Recharts for data visualization
- **Web3**: ethers.js and wagmi for blockchain interaction

## Performance Optimization

The portal implements several optimization strategies:

1. **Code Splitting**: Dynamic imports for route-based splitting
2. **Lazy Loading**: Components loaded on demand
3. **Optimistic Updates**: UI updates before blockchain confirmation
4. **Caching**: Redux persist for offline support
5. **Background Sync**: Periodic data refresh

## Getting Started

To access the management portal:

1. **Navigate** to the portal URL
2. **Connect** your Web3 wallet
3. **Sign** the authentication message
4. **Access** features based on your role

<Callout type="tip">
  <strong>First Time Users:</strong> Contact a system owner to be assigned an appropriate role before accessing the portal.
</Callout>
