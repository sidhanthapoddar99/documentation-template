# Installation Guide

This comprehensive guide covers installing and configuring Neuralock SDKs for both frontend and backend applications in production environments.

## System Requirements

Before installing Neuralock, ensure your environment meets these requirements:

### Frontend (JavaScript/TypeScript)
- **Node.js**: 18.0 or higher
- **NPM**: 8.0+ or Yarn 1.22+
- **Browser Support**: Chrome 90+, Firefox 88+, Safari 14+, Edge 90+
- **Framework Compatibility**: React 17+, Vue 3+, Angular 12+, Vanilla JS

### Backend (Python)
- **Python**: 3.8 or higher
- **pip**: 20.0 or higher
- **OS**: Linux, macOS, Windows (WSL recommended)
- **Memory**: Minimum 512MB RAM

### Blockchain Requirements
- **Web3 Wallet**: MetaMask, WalletConnect, or similar
- **Network Access**: Base, Ethereum, or compatible EVM chain
- **Gas Funds**: Small amount for session initialization

                    user_address,

                    permission_level
                )
                
                if not has_permission:
                    return jsonify({'error': 'Permission denied'}), 403
                
                return f(*args, **kwargs)
            
            return decorated_function
        return decorator

## Troubleshooting

<Callout type="warning" title="Common Installation Issues">

### Module Not Found

**Error**: `Cannot find module '@neuralock/client'`

**Solution**:
```bash
# Clear npm cache
npm cache clean --force

# Reinstall
npm install @neuralock/client
```

### TypeScript Errors

**Error**: `Cannot find type definitions`

**Solution**:
```bash
# Install types
npm install --save-dev @types/neuralock__client
```

### Network Connection Issues

**Error**: `Failed to connect to Neuralock servers`

**Solution**:
1. Check your internet connection
2. Verify RPC URL is correct
3. Ensure servers are online
4. Check firewall settings

### Smart Contract Errors

**Error**: `Contract not deployed at address`

**Solution**:
1. Verify you're on the correct network
2. Check contract address in configuration
3. Ensure contract is deployed

</Callout>

  - **Never commit private keys** - Use environment variables
  - **Use hardware wallets** in production environments
  - **Implement rate limiting** for API endpoints
  - **Enable CORS** only for trusted domains
  - **Use HTTPS** for all server communications
  - **Rotate session keys** regularly
  - **Monitor for anomalies** in encryption/decryption patterns
  - **Implement proper access controls** at application level
  - **Keep SDKs updated** to latest versions
  - **Use multi-signature wallets** for admin operations
