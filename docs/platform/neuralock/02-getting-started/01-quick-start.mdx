# Quick Start

Get started with Neuralock's distributed encryption system in under 5 minutes. This guide will walk you through creating your first encrypted application using test mode.

## Overview

Neuralock provides a revolutionary approach to data security by distributing encryption keys across multiple servers using Shamir's Secret Sharing. This quick start guide uses a simplified single-server test mode to get you up and running quickly.

## Prerequisites

Before you begin, ensure you have:

- Node.js 18+ or Python 3.8+
- A Web3 wallet (MetaMask recommended)
- Access to a blockchain testnet (Base Sepolia recommended)

<FileCollapsibleCodeBlock 
  content={`import React, { useState, useEffect } from 'react';
import { NeuralockClient } from '@neuralock/client';

function NeuralockDemo() {
  const [client, setClient] = useState(null);
  const [status, setStatus] = useState('Disconnected');
  const [encryptedData, setEncryptedData] = useState(null);
  
  // Initialize Neuralock
  const initialize = async () => {
    try {
      setStatus('Connecting...');
      
      // Connect wallet
      await window.ethereum.request({ method: 'eth_requestAccounts' });
      const provider = new ethers.providers.Web3Provider(window.ethereum);
      const signer = provider.getSigner();
      
      // Create client
      const neuralockClient = new NeuralockClient({
        applicationContract: "0x742d35Cc6634C0532925a3b844Bc9e7595f8fA32",
        signer: signer,
        servers: [{ nftId: 1 }],
        options: {
          threshold: { mode: 'single' }
        }
      });
      
      await neuralockClient.initialize();
      setClient(neuralockClient);
      setStatus('Connected');
    } catch (error) {
      setStatus('Error: ' + error.message);
    }
  };
  
  // Encrypt data
  const encryptData = async () => {
    if (!client) return;
    
    try {
      setStatus('Encrypting...');
      
      const data = { 
        message: "Hello Neuralock!", 
        timestamp: new Date().toISOString() 
      };
      
      const objectId = ethers.utils.id("demo-" + Date.now());
      const encrypted = await client.encrypt(JSON.stringify(data), objectId);
      
      setEncryptedData(encrypted);
      setStatus('Encrypted successfully!');
    } catch (error) {
      setStatus('Encryption error: ' + error.message);
    }
  };
  
  // Decrypt data
  const decryptData = async () => {
    if (!client || !encryptedData) return;
    
    try {
      setStatus('Decrypting...');
      
      const decrypted = await client.decrypt(
        encryptedData, 
        encryptedData.objectId
      );
      
      const data = JSON.parse(decrypted);
      setStatus('Decrypted: ' + data.message);
    } catch (error) {
      setStatus('Decryption error: ' + error.message);
    }
  };
  
  return (
    <div className="neuralock-demo">
      <h2>Neuralock Quick Start Demo</h2>
      <p>Status: {status}</p>
      
      <div className="actions">
        <button onClick={initialize} disabled={client !== null}>
          Initialize Neuralock
        </button>
        
        <button onClick={encryptData} disabled={!client}>
          Encrypt Data
        </button>
        
        <button onClick={decryptData} disabled={!encryptedData}>
          Decrypt Data
        </button>
      </div>
      
      {encryptedData && (
        <div className="encrypted-preview">
          <h3>Encrypted Data:</h3>
          <code>{encryptedData.ciphertext.substring(0, 50)}...</code>
        </div>
      )}
    </div>
  );
}

export default NeuralockDemo;`}
  filename="NeuralockDemo.jsx"

## Quick Reference

### Key Concepts

- **Single Server Mode**: Test mode using one server (development only)
- **Object ID**: Unique identifier for each encrypted item
- **Session**: Ephemeral connection between client and servers
- **Threshold**: Minimum servers needed to decrypt (1 in test mode)

### Common Issues

<Callout type="info" title="Troubleshooting">

**"Session expired" error**
- Sessions last 5 minutes by default
- Call `client.refreshSession()` to renew

**"Permission denied" error**
- Ensure your wallet address has access
- Check the application contract permissions

**"Server not responding" error**
- Verify the server NFT ID is correct
- Check your network connection

</Callout>
