# Callout Component

The Callout component is used to highlight important information, warnings, tips, or notes with distinctive styling and icons.

## Import

```javascript
import { Callout } from '@site/src/components/elements/Callout';
```

## Basic Usage

<Callout>
  This is a default callout with important information.
</Callout>

```jsx
<Callout>
  This is a default callout with important information.
</Callout>
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `type` | `'info' \| 'warning' \| 'success' \| 'danger' \| 'tip' \| 'note'` | `'info'` | The type of callout which determines styling and icon |
| `title` | `string` | - | Optional title displayed at the top of the callout |
| `children` | `ReactNode` | - | Content to display inside the callout |
| `className` | `string` | - | Additional CSS classes |

    <p>Follow these steps to install the package:</p>
    
    <ol>
      <li>
        <p>Install dependencies:</p>
        <pre><code className="language-bash">npm install @neuralabs/sdk</code></pre>
      </li>
      <li>

        <pre><code className="language-javascript">{`const config = {
  apiKey: 'your-api-key',
  network: 'mainnet'
};`}</code></pre>
      </li>
      <li>Initialize the client and start building!</li>
    </ol>
  </div>
</Callout>

## Styling

The Callout component uses theme-aware CSS variables:

- `--theme-callout-{type}-bg`: Background color for each type
- `--theme-callout-{type}-border`: Border color for each type
- `--theme-callout-{type}-text`: Text color for each type

## Best Practices

1. **Choose appropriate types**: Use the correct callout type for the message
2. **Keep it concise**: Callouts should highlight key information
3. **Use titles sparingly**: Only add titles when they provide additional context
4. **Don't overuse**: Too many callouts can reduce their effectiveness
5. **Accessibility**: The icons are decorative; the type and content convey meaning
