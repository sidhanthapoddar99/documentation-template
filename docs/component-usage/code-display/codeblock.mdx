# CodeBlock Components

The CodeBlock components provide enhanced code display capabilities with collapsible sections and inline code cards.

## Import


## Props Reference

### CollapsibleCodeBlock Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | Title displayed in the header |
| `description` | `string` | - | Optional description below the title |
| `language` | `string` | `'bash'` | Programming language for syntax highlighting |
| `children` | `string` | - | Code content to display |
| `defaultCollapsed` | `boolean` | `true` | Initial collapsed state |

### InlineCodeCard Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | Title displayed in the header |
| `description` | `string` | - | Optional description below the title |
| `language` | `string` | - | Programming language for syntax highlighting |
| `code` | `string` | - | Code content to display |
| `defaultExpanded` | `boolean` | `false` | Initial expanded state |

<CollapsibleCodeBlock 
  title="agent.implementation.js" 
  language="javascript"
  description="Agent implementation using the config"
>
{`import config from './agent.config.js';
import { TradingAgent } from '@neuralabs/sdk';

class AdvancedTradingBot extends TradingAgent {
  constructor() {
    super(config);
    this.initializeStrategies();
  }
  
  async executeStrategy(marketData) {
    const signals = await this.analyzeMarket(marketData);
    
    if (signals.confidence > 0.8) {
      return this.executeTrade({
        action: signals.action,
        amount: this.calculatePositionSize(signals),
        stopLoss: config.settings.stopLoss,
        takeProfit: config.settings.takeProfit
      });
    }
  }
}

export default AdvancedTradingBot;`}
</CollapsibleCodeBlock>

## Styling

The CodeBlock components use theme CSS variables:

- `--theme-code-bg`: Background color
- `--theme-code-text`: Text color
- `--theme-code-border`: Border color
- `--theme-bg-elevated`: Header background
- `--theme-text-heading`: Title color
- `--theme-text-secondary`: Description color

## Best Practices

1. **Use descriptive titles**: Help users understand what the code does
2. **Add descriptions**: Provide context for complex code snippets
3. **Choose appropriate language**: Ensure correct syntax highlighting
4. **Consider default state**: Collapse long code by default, expand short snippets
5. **Group related code**: Use multiple blocks for multi-file examples
6. **Keep code readable**: Format and indent properly
