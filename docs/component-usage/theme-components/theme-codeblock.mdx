# Theme CodeBlock Component

The Docusaurus theme CodeBlock component provides advanced syntax highlighting with language support, line numbers, and other features. This is different from the custom CodeBlock components in our elements library.

## Import

```javascript
import CodeBlock from '@theme/CodeBlock';
```

## Props Reference

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `children` | `string` | - | **Required.** The code content to display |
| `language` | `string` | `text` | Programming language for syntax highlighting |
| `title` | `string` | - | Filename or title to display above the code |
| `showLineNumbers` | `boolean` | `false` | Display line numbers |
| `metastring` | `string` | - | Additional metadata for the code block |
| `className` | `string` | - | Additional CSS classes |

## Highlighting Options

### Line Highlighting Comments

- `// highlight-next-line` - Highlight the next line
- `// highlight-start` and `// highlight-end` - Highlight a range
- `// highlight-line` - Highlight the current line

{`module trading::pool {
    use sui::object::{Self, UID};
    use sui::coin::{Self, Coin};
    use sui::balance::{Self, Balance};
    use sui::tx_context::{Self, TxContext};
    
    // highlight-start
    struct Pool<phantom X, phantom Y> has key {
        id: UID,
        balance_x: Balance<X>,
        balance_y: Balance<Y>,
        lp_supply: u64,
    }
    // highlight-end
    
    public entry fun create_pool<X, Y>(
        coin_x: Coin<X>,
        coin_y: Coin<Y>,
        ctx: &mut TxContext
    ) {
        let pool = Pool {
            id: object::new(ctx),
            balance_x: coin::into_balance(coin_x),
            balance_y: coin::into_balance(coin_y),
            lp_supply: 0,
        };
        
        // highlight-next-line
        transfer::share_object(pool);
    }
}`}
</CodeBlock>

## Best Practices

1. **Language specification**: Always specify the language for proper syntax highlighting
2. **Meaningful titles**: Use descriptive filenames in the title prop
3. **Line numbers**: Enable for longer code blocks or when referencing specific lines
4. **Highlighting**: Use sparingly to draw attention to important parts
5. **Code formatting**: Ensure proper indentation and formatting before displaying
6. **String escaping**: Use template literals with curly braces for JSX content

## Differences from Custom CodeBlock

| Feature | Theme CodeBlock | Custom CodeBlock (Elements) |
|---------|----------------|---------------------------|
| Import | `@theme/CodeBlock` | `@site/src/components/elements/CodeBlock` |
| Syntax Highlighting | Built-in Prism.js | Built-in Prism.js |
| Collapsible | No | Yes (CollapsibleCodeBlock) |
| Inline Card | No | Yes (InlineCodeCard) |
| Line Highlighting | Yes | No |
| Line Numbers | Yes | Limited |
| Title | Yes | Yes |
| Live Editor | With plugin | No |

Choose the theme CodeBlock for:
- Standard documentation code examples
- Line highlighting needs
- Line number display
- MDX automatic transformation

Choose custom CodeBlock components for:
- Collapsible code sections
- Inline expandable code cards
- Custom styling needs
