# Tabs Component

The Tabs component allows you to organize content into multiple tabs, perfect for showing different code examples, platform-specific instructions, or organizing related content.

## Import

```javascript
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
```

## Query String Parameters

Tabs can be controlled via URL query parameters:

<Tabs queryString="config-type">
  <TabItem value="basic" label="Basic Config">
    ```javascript
    const config = {
      apiKey: 'your-key',
      network: 'mainnet'
    };
    ```
  </TabItem>
  <TabItem value="advanced" label="Advanced Config">
    ```javascript
    const config = {
      apiKey: 'your-key',
      network: 'mainnet',
      timeout: 30000,
      retries: 3,
      cache: {
        enabled: true,
        ttl: 60000
      },
      logging: {
        level: 'debug',
        format: 'json'
      }
    };
    ```
  </TabItem>
</Tabs>

```mdx
<Tabs queryString="config-type">
  <TabItem value="basic" label="Basic Config">
    // Basic configuration
  </TabItem>
  <TabItem value="advanced" label="Advanced Config">
    // Advanced configuration
  </TabItem>
</Tabs>
```

Now users can link directly to a specific tab: `?config-type=advanced`

## Props Reference

### Tabs Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `defaultValue` | `string` | First tab | The default tab to display |
| `values` | `Array` | - | Array of tab values (auto-detected from children) |
| `groupId` | `string` | - | Synchronize tab selection between tab groups |
| `className` | `string` | - | Additional CSS classes |
| `queryString` | `string \| boolean` | `false` | Enable URL query string support |
| `lazy` | `boolean` | `false` | Lazy load tab content |

### TabItem Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `value` | `string` | - | **Required.** The tab identifier |
| `label` | `string` | - | Tab button label |
| `default` | `boolean` | `false` | Mark as default tab |
| `hidden` | `boolean` | `false` | Hide this tab |
| `className` | `string` | - | Additional CSS classes |
| `attributes` | `object` | - | Additional HTML attributes |

### Framework Integration

<Tabs groupId="framework">
  <TabItem value="react" label="React">
    ```jsx
    import { NeuraLabsProvider, useNeuraLabs } from '@neuralabs/react';
    
    function App() {
      return (
        <NeuraLabsProvider config={config}>
          <Dashboard />
        </NeuraLabsProvider>
      );
    }
    
    function Dashboard() {
      const { agents, createAgent } = useNeuraLabs();
      
      return (
        <div>
          {agents.map(agent => (
            <AgentCard key={agent.id} agent={agent} />
          ))}
        </div>
      );
    }
    ```
  </TabItem>
  <TabItem value="vue" label="Vue">
    ```vue
    <template>
      <div>
        <agent-card 
          v-for="agent in agents" 
          :key="agent.id"
          :agent="agent"
        />
      </div>
    </template>
    
    <script>
    import { useNeuraLabs } from '@neuralabs/vue';
    
    export default {
      setup() {
        const { agents, createAgent } = useNeuraLabs();
        
        return {
          agents,
          createAgent
        };
      }
    };
    </script>
    ```
  </TabItem>
  <TabItem value="angular" label="Angular">
    ```typescript
    import { Component } from '@angular/core';
    import { NeuraLabsService } from '@neuralabs/angular';
    
    @Component({
      selector: 'app-dashboard',
      template: `
        <div *ngFor="let agent of agents$ | async">
          <agent-card [agent]="agent"></agent-card>
        </div>
      `
    })
    export class DashboardComponent {
      agents$ = this.neuralabs.getAgents();
      
      constructor(private neuralabs: NeuraLabsService) {}
    }
    ```
  </TabItem>
</Tabs>

## Best Practices

1. **Use meaningful values**: Tab values should be descriptive and URL-friendly
2. **Consistent grouping**: Use `groupId` for related tab groups
3. **Default selection**: Always specify a sensible default tab
4. **Mobile consideration**: Test tab behavior on mobile devices
5. **Content organization**: Keep tab content focused and concise
6. **Accessibility**: Tabs are keyboard navigable by default
