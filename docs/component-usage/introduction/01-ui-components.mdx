## Quick Start

### Installation

All components are pre-installed in the documentation system. To use them in your MDX files:

**Custom Components:**
```javascript
import { ComponentName } from '@site/src/components/elements/ComponentName';
```

**Theme Components:**
```javascript
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import CodeBlock from '@theme/CodeBlock';
```

### Basic Usage Example

```mdx
---
title: My Documentation Page
---

import { Card, CardHeader, CardTitle } from '@site/src/components/elements/Card';
import { Callout } from '@site/src/components/elements/Callout';
import Tabs from '@theme/Tabs';
import TabItem from '@theme/TabItem';
import CodeBlock from '@theme/CodeBlock';

# My Page

<Card>
  <CardHeader>
    <CardTitle>Welcome</CardTitle>
  </CardHeader>
  <p>This is a card component.</p>
</Card>

<Callout type="info">
  This is an informational callout.
</Callout>

<Tabs>
  <TabItem value="js" label="JavaScript">
    <CodeBlock language="javascript">
      {`const greeting = "Hello World";`}
    </CodeBlock>
  </TabItem>
  <TabItem value="py" label="Python">
    <CodeBlock language="python">
      {`greeting = "Hello World"`}
    </CodeBlock>
  </TabItem>
</Tabs>
```

### Using Theme Colors

All components use the centralized color theme system:

```css
.my-component {
  background: var(--theme-bg-surface);
  color: var(--theme-text-primary);
  border: 1px solid var(--theme-border-default);
}
```

### Adding New Components

1. Create component in `/src/components/elements/`
2. Use theme CSS variables for styling
3. Export from index.js
4. Document in this guide
5. Test in both light and dark modes

## Best Practices

### 1. Consistent Styling
- Always use theme CSS variables
- Follow existing component patterns
- Maintain consistent spacing

### 2. Accessibility
- Use semantic HTML
- Include ARIA labels where needed
- Ensure keyboard navigation works

### 3. Performance
- Lazy load heavy components
- Optimize images and icons
- Minimize CSS bundle size

### 4. Documentation
- Provide clear examples
- Document all props
- Include common use cases
