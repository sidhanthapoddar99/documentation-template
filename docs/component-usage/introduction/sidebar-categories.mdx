### File Naming Conventions

- Use `.mdx` extension for Markdown files with JSX support
- Use `.md` extension for pure Markdown files
- Category folders should have descriptive names

## Category Types

### 1. Non-Clickable Headers (Label Only)

Used for section headers that organize content but aren't clickable.

<CodeBlock language="json" title="_category_.json">
{`{
  "label": "SECTION HEADER",
  "position": 1,
  "className": "sidebar-header",
  "collapsible": false
}`}
</CodeBlock>

<Callout type="info">
This is the pattern used in this documentation for headers like "CARDS & CONTAINERS".
</Callout>

### 2. Clickable Category with Link to First Doc

Category that navigates to a specific document when clicked.

<CodeBlock language="json" title="_category_.json">
{`{
  "label": "Getting Started",
  "position": 1,
  "link": {
    "type": "doc",
    "id": "introduction/getting-started"
  }
}`}
</CodeBlock>

### 3. Generated Index Page

Creates an auto-generated index page listing all items in the category.

<CodeBlock language="json" title="_category_.json">
{`{
  "label": "Components",
  "position": 2,
  "link": {
    "type": "generated-index",
    "title": "Component Library",
    "description": "All available components",
    "slug": "/components",
    "keywords": ["components", "ui"]
  }
}`}
</CodeBlock>

### 4. External Link Category

Category that links to an external URL.

<CodeBlock language="json" title="_category_.json">
{`{
  "label": "API Reference",
  "position": 3,
  "link": {
    "type": "url",
    "href": "https://api.example.com"
  }
}`}
</CodeBlock>

### 5. Collapsible Category

Standard collapsible category without a link.

<CodeBlock language="json" title="_category_.json">
{`{
  "label": "Advanced Topics",
  "position": 4,
  "collapsible": true,
  "collapsed": true
}`}
</CodeBlock>

## Document Front Matter

### Basic Document Configuration

<CodeBlock language="yaml" title="document.mdx">
{`---
id: unique-doc-id          # Required: unique identifier
title: Document Title      # Page title
description: SEO description
sidebar_position: 1        # Position in sidebar
sidebar_label: Short Title # Optional: different sidebar text
slug: /custom-url         # Optional: custom URL path
tags: [tutorial, basic]   # Optional: document tags
---`}
</CodeBlock>

### Advanced Front Matter Options

<CodeBlock language="yaml" title="advanced-doc.mdx">
{`---
id: advanced-doc
title: Advanced Configuration
hide_title: false          # Hide the title
hide_table_of_contents: false  # Hide TOC
sidebar_class_name: custom-item # Custom sidebar CSS
pagination_label: Custom Label  # Custom pagination text
custom_edit_url: https://github.com/... # Custom edit URL
keywords: [seo, keywords]      # SEO keywords
image: ./thumbnail.png         # Social media image
---`}
</CodeBlock>

## Best Practices

<Callout type="success" title="Do's">
- Use consistent naming conventions
- Keep category names concise and descriptive
- Organize related content together
- Use `sidebar_position` for explicit ordering
- Add descriptions to generated index pages
</Callout>

<Callout type="warning" title="Don'ts">
- Avoid deeply nested structures (>3 levels)
- Don't use special characters in file names
- Avoid duplicate IDs across documents
- Don't mix different organizational patterns
</Callout>
