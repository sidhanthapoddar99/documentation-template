# Multiple Documentation Sections

When your project grows, you might need separate documentation sections with their own sidebars and routing. This is where multiple docs configuration becomes essential.

## When to Use Multiple Docs

<Tabs>
<TabItem value="single-vs-multiple" label="Single vs Multiple">

**Single Documentation (Default):**
- All content in one sidebar
- Routes like `/docs/section/page`
- Good for: Simple projects, unified documentation

**Multiple Documentation Sections:**
- Separate sidebars for each section
- Routes like `/api/page`, `/guides/page`, `/docs/page`
- Good for: Complex projects, API + guides, different audiences

</TabItem>
<TabItem value="use-cases" label="Common Use Cases">

**Typical Multiple Docs Scenarios:**
- **API Reference** + **User Guides**
- **Developer Docs** + **End User Docs**
- **Different Product Modules**
- **Versioned Documentation**
- **Multi-language Sections**

</TabItem>
</Tabs>

### Key Configuration Options

<Callout type="info">
**Important**: Each docs plugin instance must have a unique `id`. This ID is used to reference the plugin in navigation and other configurations.
</Callout>

<Tabs>
<TabItem value="required" label="Required Options">

**Essential Settings:**
- **`id`**: Unique identifier for this docs section
- **`path`**: Folder containing the documentation files
- **`routeBasePath`**: URL path prefix for this section
- **`sidebarPath`**: Path to sidebar configuration file

</TabItem>
<TabItem value="optional" label="Optional Options">

**Additional Settings:**
- **`editUrl`**: Base URL for "Edit this page" links
- **`showLastUpdateTime`**: Display last modified date
- **`showLastUpdateAuthor`**: Show last editor
- **`customProps`**: Custom data accessible in components

</TabItem>
</Tabs>

## Sidebar Configuration

Each docs section needs its own sidebar configuration file:

<Tabs>
<TabItem value="api-sidebar" label="API Sidebar">

```javascript
// sidebarsApi.js
module.exports = {
  apiSidebar: [
    'introduction',
    'authentication',
    {
      type: 'category',
      label: 'Endpoints',
      items: [
        'endpoints/users',
        'endpoints/orders',
      ],
    },
    {
      type: 'autogenerated',
      dirName: 'reference',
    },
  ],
};
```

</TabItem>

## URL Structure

With multiple docs configuration, your URLs will be:

<Tabs>
<TabItem value="url-examples" label="URL Examples">

**File → URL Mapping:**
- `docs/main/intro.md` → `/docs/intro`
- `docs/api/introduction.md` → `/api/introduction`
- `docs/guides/getting-started.md` → `/guides/getting-started`

</TabItem>
<TabItem value="benefits" label="Benefits">

**Advantages of This Structure:**
- **Clear Separation**: Each section has its own URL space
- **Independent Navigation**: Separate sidebars for focused browsing
- **Flexible Routing**: Customize URL paths for each section
- **Better SEO**: Clear content categorization

</TabItem>
</Tabs>

## Best Practices

<Callout type="success">
**Multiple Docs Best Practices:**

1. **Meaningful IDs**: Use descriptive plugin IDs like 'api', 'guides', 'tutorials'
2. **Consistent Structure**: Keep similar folder organization across sections
3. **Clear Navigation**: Make it obvious which section users are in
4. **Cross-references**: Link between sections when relevant
5. **Separate Audiences**: Design each section for its intended users
</Callout>

## Common Patterns

### API Documentation + User Guides
```javascript
// Perfect for SaaS products
plugins: [
  ['@docusaurus/plugin-content-docs', {
    id: 'api',
    path: 'docs/api',
    routeBasePath: 'api',
    // ... API-specific config
  }],
  ['@docusaurus/plugin-content-docs', {
    id: 'guides', 
    path: 'docs/guides',
    routeBasePath: 'guides',
    // ... Guides-specific config
  }],
]
```

### Product Modules
```javascript
// For complex products with multiple modules
plugins: [
  ['@docusaurus/plugin-content-docs', {
    id: 'module-a',
    path: 'docs/module-a',
    routeBasePath: 'module-a',
  }],
  ['@docusaurus/plugin-content-docs', {
    id: 'module-b',
    path: 'docs/module-b', 
    routeBasePath: 'module-b',
  }],
]
```

<Callout type="warning">
**Important**: Remember to update your sidebar references in navigation items to use the correct `docsPluginId` for each section.
</Callout>

This setup gives you the flexibility to organize complex documentation while maintaining clean, focused user experiences for each section.
