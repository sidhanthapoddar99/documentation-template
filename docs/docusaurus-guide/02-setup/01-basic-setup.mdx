## Three Types of Pages

Docusaurus supports three main types of pages, each serving a different purpose:

<Tabs>
<TabItem value="docs" label="Documentation Pages">

**Purpose**: Structured documentation with sidebars and navigation
**Location**: `docs/` folder
**Features**:
- Automatic sidebar generation
- Version control support
- Category organization
- Search integration

**Example Route**: `docs/intro.md` → `/docs/intro`

</TabItem>
<TabItem value="blog" label="Blog Pages">

**Purpose**: Time-based content like announcements and updates
**Location**: `blog/` folder  
**Features**:
- RSS feed generation
- Author profiles
- Tag system
- Archive pages

**Example Route**: `blog/2024-01-01-hello.md` → `/blog/hello`

</TabItem>
<TabItem value="pages" label="Custom Pages">

**Purpose**: Standalone pages like homepage, about, contact
**Location**: `src/pages/` folder
**Features**:
- Full React component flexibility
- Custom layouts
- Direct routing

**Example Route**: `src/pages/about.js` → `/about`

</TabItem>
</Tabs>

## Configuration Overview

The main configuration happens in `docusaurus.config.js`. This file controls:

- **Site metadata** (title, description, URL)
- **Theme configuration** (navbar, footer, colors)
- **Plugin settings** (docs, blog, search)
- **Deployment settings** (GitHub Pages, etc.)

<Callout type="tip">
**Directory-Based Routing**: One of Docusaurus's key features is that your file and folder structure automatically determines your site's URL structure. No need to manually configure routes!
</Callout>

## Development Workflow

1. **Create Content**: Add Markdown files to `docs/` or `blog/`
2. **Customize**: Modify `docusaurus.config.js` for site settings
3. **Style**: Add custom CSS in `src/css/custom.css`
4. **Preview**: Run `npm start` to see changes in real-time
5. **Build**: Run `npm run build` to generate static files
6. **Deploy**: Upload the `build/` folder to your hosting provider
