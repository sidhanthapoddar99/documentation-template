# Docs Configuration

Docusaurus provides flexible configuration options for documentation and blog content. You can set up a single documentation section or multiple sections depending on your needs.

## Single Documentation Section

The most common setup uses the `classic` preset which includes docs, blog, and theme configuration:

### Configuration Options Explained

<Tabs>
<TabItem value="docs" label="Docs Options">

**Key Documentation Settings:**

- **`routeBasePath`**: URL path for docs (e.g., 'docs' → `/docs`)
- **`sidebarPath`**: Path to sidebar configuration file
- **`editUrl`**: Base URL for "Edit this page" links
- **`showLastUpdateTime`**: Display last modified date
- **`showLastUpdateAuthor`**: Display last editor

</TabItem>
<TabItem value="blog" label="Blog Options">

**Key Blog Settings:**

- **`showReadingTime`**: Calculate and display reading time
- **`editUrl`**: Base URL for "Edit this page" links
- **`blogSidebarTitle`**: Title for blog sidebar
- **`blogSidebarCount`**: Number of recent posts to show

</TabItem>
<TabItem value="theme" label="Theme Options">

**Theme Settings:**

- **`customCss`**: Path to custom CSS file
- **`colorMode`**: Dark/light mode settings
- **`prism`**: Code syntax highlighting theme

</TabItem>
</Tabs>

## How Routing Works

With the configuration above:

<Tabs>
<TabItem value="docs-routes" label="Documentation Routes">

**File Structure → URLs:**
- `docs/intro.md` → `/docs/intro`
- `docs/tutorial/basics.md` → `/docs/tutorial/basics`
- `docs/api/reference.md` → `/docs/api/reference`

**Features:**
- Automatic sidebar generation
- Search integration
- Navigation between docs

</TabItem>
<TabItem value="blog-routes" label="Blog Routes">

**File Structure → URLs:**
- `blog/2024-01-01-hello.md` → `/blog/hello`
- `blog/2024-02-15-update.md` → `/blog/update`

**Features:**
- Chronological listing
- RSS feed generation
- Tag and archive pages

</TabItem>
<TabItem value="page-routes" label="Custom Page Routes">

**File Structure → URLs:**
- `src/pages/index.js` → `/` (homepage)
- `src/pages/about.js` → `/about`
- `src/pages/contact.js` → `/contact`

**Features:**
- Full React component flexibility
- Custom layouts and styling

</TabItem>
</Tabs>

## Configuration Best Practices

<Callout type="tip">
**Recommended Settings:**
- Always set `editUrl` to enable collaborative editing
- Enable `showLastUpdateTime` for documentation freshness
- Use descriptive `routeBasePath` names for clarity
- Configure RSS feeds for blogs to improve discoverability
</Callout>

## Sidebar Configuration

The `sidebarPath` option points to your sidebar configuration file (usually `sidebars.js`):

```javascript
// sidebars.js
module.exports = {
  docsSidebar: [
    'intro',
    {
      type: 'category',
      label: 'Tutorial',
      items: ['tutorial/basics', 'tutorial/advanced'],
    },
    {
      type: 'autogenerated',
      dirName: 'api', // Generate from /docs/api
    },
  ],
};
```

<Callout type="success">
**Auto-generated Sidebars**: Use `type: 'autogenerated'` to automatically create sidebar items from your folder structure. This is perfect for large documentation sections!
</Callout>

## Common Patterns

### Documentation-First Site
```javascript
docs: {
  routeBasePath: '/', // Docs at root URL
  sidebarPath: require.resolve('./sidebars.js'),
}
```

### Multi-Language Setup
```javascript
i18n: {
  defaultLocale: 'en',
  locales: ['en', 'fr', 'es'],
}
```

### Development vs Production
```javascript
docs: {
  editUrl: process.env.NODE_ENV === 'development' 
    ? undefined 
    : 'https://github.com/myorg/docs/edit/main/',
}
```

This configuration approach gives you a solid foundation for organizing your content while maintaining flexibility for future expansion.
