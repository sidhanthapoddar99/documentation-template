# Common Plugins

Docusaurus provides a rich ecosystem of plugins that add functionality to your documentation site. Here are the most commonly used plugins and their configurations.

## Analytics Plugins

### Google Analytics

<Callout type="info">
**Google Analytics Integration**: Track page views, user behavior, and site performance with Google Analytics.
</Callout>

<FileCollapsibleCodeBlock content={GoogleAnalyticsCode} filename="google-analytics-plugin.js" />

**Key Options:**
- `trackingID`: Your Google Analytics tracking ID (required)
- `anonymizeIP`: Anonymize IP addresses for privacy compliance
- `gtag`: Use gtag.js instead of analytics.js (recommended)

### Google Tag Manager

```javascript
plugins: [
  [
    '@docusaurus/plugin-google-gtag',
    {
      trackingID: 'GTM-XXXXXXX',
      anonymizeIP: true,
    },
  ],
]
```

## Progressive Web App (PWA)

<Callout type="feature">
**PWA Features**: Transform your documentation into a Progressive Web App with offline support and app-like experience.
</Callout>

<FileCollapsibleCodeBlock content={PWACode} filename="pwa-plugin.js" />

**PWA Benefits:**
- **Offline Access** - Users can browse docs without internet
- **App Installation** - Can be installed like a native app
- **Fast Loading** - Cached resources load instantly
- **Push Notifications** - Notify users of updates

**Required Files:**
- `static/manifest.json` - PWA manifest
- `static/img/` - App icons in various sizes

## Plugin Configuration Patterns

### Environment-Specific Plugins

<Callout type="tip">
**Environment Configuration**: Use environment variables to configure plugins differently for development and production.
</Callout>

```javascript
const isDev = process.env.NODE_ENV === 'development';

module.exports = {
  plugins: [
    // Only in production
    !isDev && [
      '@docusaurus/plugin-google-analytics',
      { trackingID: 'UA-XXXXXXXX-X' }
    ],
    
    // Only in development
    isDev && [
      '@docusaurus/plugin-debug',
      { enabled: true }
    ],
    
    // Always enabled
    '@docusaurus/plugin-sitemap',
  ].filter(Boolean),
};
```

### Conditional Plugin Loading

```javascript
const plugins = [
  '@docusaurus/plugin-content-docs',
  '@docusaurus/plugin-content-blog',
];

// Add analytics only if tracking ID is provided
if (process.env.GA_TRACKING_ID) {
  plugins.push([
    '@docusaurus/plugin-google-analytics',
    { trackingID: process.env.GA_TRACKING_ID }
  ]);
}

module.exports = {
  plugins,
};
```

## Plugin Installation

### NPM Installation

```bash
# Official plugins
npm install @docusaurus/plugin-google-analytics

# Community plugins
npm install docusaurus-plugin-search-local

# Development dependencies
npm install --save-dev @docusaurus/plugin-debug
```

### Yarn Installation

```bash
# Official plugins
yarn add @docusaurus/plugin-google-analytics

# Community plugins
yarn add docusaurus-plugin-search-local

# Development dependencies
yarn add --dev @docusaurus/plugin-debug
```

<Callout type="warning">
**Plugin Compatibility**: Always check plugin compatibility with your Docusaurus version. Some plugins may require specific versions or have breaking changes.
</Callout>

## Plugin Troubleshooting

### Common Issues

1. **Plugin Not Loading**
   - Check plugin installation
   - Verify configuration syntax
   - Ensure proper plugin naming

2. **Build Errors**
   - Review plugin options
   - Check for conflicting plugins
   - Validate required files exist

3. **Runtime Errors**
   - Check browser console
   - Verify plugin dependencies
   - Test in development mode

### Debug Mode

Enable debug mode to troubleshoot plugin issues:

```bash
DEBUG=docusaurus* npm start
```

This comprehensive overview covers the most popular Docusaurus plugins and their common use cases. Choose plugins based on your specific needs and always test thoroughly before deploying to production.
