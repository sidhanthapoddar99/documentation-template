# <img src="/img/icons/terminal.svg" width="32" height="32" style={{ verticalAlign: 'middle', marginRight: '12px' }} />Walrus CLI Usage

Command-line interface for direct interaction with Walrus storage, ideal for development, testing, and automation.

## Overview

The Walrus CLI provides a powerful command-line interface for storing and retrieving data from the Walrus network. It's perfect for developers who prefer terminal-based workflows, need to automate storage operations, or want to quickly test Walrus functionality.

<div className="card padding--md mt-4">
  <h3><img src="/img/icons/info.svg" width="20" height="20" style={{ verticalAlign: 'middle', marginRight: '8px' }} />CLI Features</h3>
  <ul>
    <li><strong>Direct Storage Operations:</strong> Upload and download files without writing code</li>
    <li><strong>Wallet Integration:</strong> Built-in SUI wallet support for paying storage fees</li>
    <li><strong>Batch Operations:</strong> Process multiple files efficiently</li>
    <li><strong>Configuration Management:</strong> Save and reuse network settings</li>
    <li><strong>Progress Tracking:</strong> Real-time upload/download progress</li>
  </ul>
</div>

## Installation

### Binary Installation

## Basic Operations

### Storing Files

<CollapsibleCodeBlock
  title="Upload Files to Walrus"
  description="Store single or multiple files with various options"
  language="bash"
  defaultCollapsed={false}
>
{`# Store a single file (default 5 epochs)
walrus store myfile.pdf

# Store with custom epochs
walrus store myfile.pdf --epochs 30

# Store with progress bar
walrus store large-file.zip --progress

# Store multiple files
walrus store file1.txt file2.json file3.pdf --epochs 10

# Store entire directory
walrus store ./my-directory --recursive

# Store with custom metadata
walrus store data.json --metadata '{"type":"config","version":"1.0"}'

# Store and make deletable
walrus store temp-file.txt --deletable --epochs 1

# Dry run (estimate cost without storing)
walrus store large-file.bin --dry-run

# Example output:

Estimated cost: 0.5 WAL

Success! Stored with blob ID: 2g4h5j6k8l9m0n2p3q4r5s6t7u8v9w0x1y2z3a4b5c6d
Storage cost: 0.48 WAL
Transaction: https://suiscan.xyz/testnet/tx/0x...`}
</CollapsibleCodeBlock>

## Advanced Usage

### Batch Operations

<CollapsibleCodeBlock
  title="Batch File Operations"
  description="Process multiple files efficiently with batch commands"
  language="bash"
  defaultCollapsed={false}
>
{`# Store all files matching pattern
walrus store *.json --epochs 30

# Store with manifest generation
walrus store ./dataset/* --generate-manifest manifest.json

# Batch download from manifest
walrus read-manifest manifest.json --output-dir ./restored

# Example manifest format:
{
  "version": "1.0",
  "created": "2025-01-25T10:30:00Z",
  "files": [
    {
      "name": "file1.json",
      "blob_id": "2g4h5j6k8l9m0n2p3q4r5s6t7u8v9w0x1y2z3a4b5c6d",
      "size": 1024,
      "hash": "sha256:..."
    },
    {
      "name": "file2.json",
      "blob_id": "3h5i6j7k8l9m0n2p3q4r5s6t7u8v9w0x1y2z3a4b5c6d",
      "size": 2048,
      "hash": "sha256:..."
    }
  ]
}

# Process files in parallel
walrus store ./data/*.csv --parallel 4 --epochs 50

# Batch operations with confirmation
walrus store ./important/* --epochs 100 --confirm

# Create and execute batch script
cat > batch-upload.txt << EOF
store file1.txt --epochs 10
store file2.pdf --epochs 30
store file3.zip --epochs 50
EOF

walrus batch < batch-upload.txt`}
</CollapsibleCodeBlock>

### Scripting and Automation

<CollapsibleCodeBlock
  title="Automation Scripts"
  description="Integrate Walrus CLI into your automation workflows"
  language="bash"
  defaultCollapsed={false}
>
{`#!/bin/bash
# backup-to-walrus.sh - Automated backup script

set -e

# Configuration
BACKUP_DIR="/var/backups"
EPOCHS=90
MANIFEST_FILE="$HOME/.walrus/backup-manifest.json"
LOG_FILE="/var/log/walrus-backup.log"

# Function to log messages
log() {
    echo "[$(date '+%Y-%m-%d %H:%M:%S')] $1" | tee -a "$LOG_FILE"
}

# Create backup archive
log "Creating backup archive..."
BACKUP_FILE="backup-$(date +%Y%m%d-%H%M%S).tar.gz"
tar -czf "/tmp/$BACKUP_FILE" -C "$BACKUP_DIR" .

# Calculate hash for verification
HASH=$(sha256sum "/tmp/$BACKUP_FILE" | cut -d' ' -f1)

# Upload to Walrus
log "Uploading to Walrus..."
BLOB_ID=$(walrus store "/tmp/$BACKUP_FILE" \
    --epochs $EPOCHS \
    --metadata "{\"type\":\"backup\",\"date\":\"$(date -u +%Y-%m-%dT%H:%M:%SZ)\",\"hash\":\"$HASH\"}" \
    --json | jq -r '.blob_id')

if [ -z "$BLOB_ID" ]; then
    log "ERROR: Upload failed"
    exit 1
fi

log "Successfully uploaded: $BLOB_ID"

# Update manifest
jq ". + [{
    \"date\": \"$(date -u +%Y-%m-%dT%H:%M:%SZ)\",
    \"blob_id\": \"$BLOB_ID\",
    \"size\": $(stat -c%s "/tmp/$BACKUP_FILE"),
    \"hash\": \"$HASH\",
    \"expires\": \"$(date -u -d "+$EPOCHS days" +%Y-%m-%dT%H:%M:%SZ)\"
}]" "$MANIFEST_FILE" > "$MANIFEST_FILE.tmp" && mv "$MANIFEST_FILE.tmp" "$MANIFEST_FILE"

# Cleanup
rm -f "/tmp/$BACKUP_FILE"

# Prune old backups (keep last 10)
BACKUP_COUNT=$(jq '. | length' "$MANIFEST_FILE")
if [ "$BACKUP_COUNT" -gt 10 ]; then
    jq '.[1:]' "$MANIFEST_FILE" > "$MANIFEST_FILE.tmp" && mv "$MANIFEST_FILE.tmp" "$MANIFEST_FILE"
fi

log "Backup complete"

# Send notification (optional)
# curl -X POST https://api.pushover.net/1/messages.json \
#     -d "token=YOUR_APP_TOKEN" \
#     -d "user=YOUR_USER_KEY" \
#     -d "message=Walrus backup completed: $BLOB_ID"`}
</CollapsibleCodeBlock>

### Integration Examples

<CollapsibleCodeBlock
  title="CI/CD Integration"
  description="Use Walrus CLI in continuous integration pipelines"
  language="yaml"
  defaultCollapsed={true}
>
{`# GitHub Actions example
name: Deploy to Walrus

on:
  push:
    branches: [main]
  release:
    types: [published]

jobs:
  deploy:
    runs-on: ubuntu-latest
    
    steps:
    - uses: actions/checkout@v3
    
    - name: Install Walrus CLI
      run: |
        wget https://github.com/MystenLabs/walrus-sites/releases/latest/download/walrus-x86_64-unknown-linux-gnu
        chmod +x walrus-x86_64-unknown-linux-gnu
        sudo mv walrus-x86_64-unknown-linux-gnu /usr/local/bin/walrus
        walrus --version
    
    - name: Configure Walrus
      env:
        WALRUS_PRIVATE_KEY: \${{ secrets.WALRUS_PRIVATE_KEY }}
      run: |
        mkdir -p ~/.walrus
        echo "\${{ secrets.WALRUS_CONFIG }}" > ~/.walrus/config.yaml
        echo "$WALRUS_PRIVATE_KEY" > ~/.walrus/wallet.key
        chmod 600 ~/.walrus/wallet.key
    
    - name: Build artifacts
      run: |
        npm ci
        npm run build
        tar -czf dist.tar.gz dist/
    
    - name: Upload to Walrus
      id: upload
      run: |
        BLOB_ID=$(walrus store dist.tar.gz \
          --epochs 90 \
          --metadata '{"version":"\${{ github.sha }}","branch":"\${{ github.ref_name }}"}' \
          --json | jq -r '.blob_id')
        
        echo "blob_id=$BLOB_ID" >> $GITHUB_OUTPUT
        echo "Uploaded to Walrus: $BLOB_ID"
    
    - name: Update deployment record
      run: |
        echo "{
          \"version\": \"\${{ github.sha }}\",
          \"blob_id\": \"\${{ steps.upload.outputs.blob_id }}\",
          \"timestamp\": \"$(date -u +%Y-%m-%dT%H:%M:%SZ)\",
          \"download_url\": \"https://aggregator.walrus-testnet.walrus.space/v1/\${{ steps.upload.outputs.blob_id }}\"
        }" > deployment.json
        
    - name: Create release asset
      if: github.event_name == 'release'
      uses: actions/upload-release-asset@v1
      env:
        GITHUB_TOKEN: \${{ secrets.GITHUB_TOKEN }}
      with:
        upload_url: \${{ github.event.release.upload_url }}
        asset_path: deployment.json
        asset_name: walrus-deployment.json
        asset_content_type: application/json`}
</CollapsibleCodeBlock>

## Security Best Practices

<div className="card padding--md mt-4">
  <h3><img src="/img/icons/security.svg" width="20" height="20" style={{ verticalAlign: 'middle', marginRight: '8px' }} />CLI Security</h3>
  
  **Wallet Security:**
  - Never share your recovery phrase
  - Use hardware wallets for mainnet
  - Regularly backup wallet configuration
  - Use separate wallets for testing
  
  **Data Security:**
  - Always encrypt sensitive data before uploading
  - Verify blob integrity after download
  - Use secure channels for blob ID sharing
  - Implement access controls at application level
  
  **Operational Security:**
  - Restrict CLI access on shared systems
  - Use environment variables for sensitive config
  - Audit CLI operations regularly
  - Monitor unexpected storage costs
</div>
