# Card Components

Cards provide visually distinct containers for grouping related content. The framework includes three card variants for different use cases.

## Card

The default Card component displays content in a bordered container with optional title and icon.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `title` | `string` | - | Card heading text |
| `icon` | `string` | - | Emoji or icon for the header |
| `href` | `string` | - | Makes the card clickable and links to this URL |

### Basic Usage

```astro
<Card title="Getting Started">
  Learn how to set up your documentation site.
</Card>
```

## CardGrid

CardGrid arranges multiple cards in a responsive grid layout.

### Props

| Prop | Type | Default | Description |
|------|------|---------|-------------|
| `cols` | `2 \| 3 \| 4` | `3` | Number of columns at desktop width |
| `gap` | `'sm' \| 'md' \| 'lg'` | `'md'` | Spacing between cards |

### Basic Usage

```astro
<CardGrid cols={3}>
  <Card title="Feature 1">Description of feature 1.</Card>
  <Card title="Feature 2">Description of feature 2.</Card>
  <Card title="Feature 3">Description of feature 3.</Card>
</CardGrid>
```

### Two Column Grid

```astro
<CardGrid cols={2} gap="lg">
  <Card title="Left Card">Content on the left.</Card>
  <Card title="Right Card">Content on the right.</Card>
</CardGrid>
```

### Responsive Behavior

- **Mobile (< 640px)**: Single column
- **Tablet (640px - 1024px)**: 2 columns
- **Desktop (> 1024px)**: Full column count (2, 3, or 4)

## Styling

All card components use CSS variables for theming:

- `--theme-card-bg` - Card background color
- `--theme-card-border` - Card border color
- `--color-brand-primary` - Hover accent color
- `--theme-shadow-md` - Hover shadow effect

Cards automatically adapt to light and dark modes.
