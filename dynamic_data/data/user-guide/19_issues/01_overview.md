# Issues

*This section is under construction as part of the phase-2 docs-update.*

The issues content type is an AI-native tracker where each issue lives as a folder in your repo, complete with subtasks, notes, comments, and an agent-log. Everything is markdown, everything is human-readable, and everything is writable by a coding agent.

Scope to be covered:

- Purpose + why it's AI-native
- Folder-per-item structure (`YYYY-MM-DD-<slug>/`)
- `settings.json` — per-issue metadata + tracker-wide vocabulary
- Sub-docs (subtasks / notes / agent-log / comments) and their URLs
- Views, filters, and presets
