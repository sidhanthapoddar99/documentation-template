# Assets

The `assets` folder allows you to organize and embed external files like code snippets, images, and videos in your documentation.

## Assets Folder

Every documentation folder can have an `assets` subfolder:

```
docs/
├── 01_getting-started/
│   ├── settings.json
│   ├── 01_overview.mdx
│   ├── 02_installation.mdx
│   └── assets/
│       ├── code/
│       │   ├── example.py
│       │   └── config.yaml
│       ├── images/
│       │   └── architecture.png
│       └── videos/
│           └── demo.mp4
```

> **Important:** The `assets` folder is **excluded from sidebar indexing**. It won't appear as a navigation item.

## Supported File Types

| Category | Extensions |
|----------|------------|
| Code | `.py`, `.js`, `.ts`, `.cpp`, `.java`, `.go`, `.rs`, `.bash`, `.sh`, `.yaml`, `.json`, `.sql` |
| Markdown | `.md` |
| Images | `.png`, `.jpg`, `.jpeg`, `.gif`, `.svg`, `.webp` |
| Videos | `.mp4`, `.webm`, `.mov` |

## Embedding Syntax

Use the `[[type, path]]` syntax to embed assets directly in your documentation:

```markdown
[[type, ./path/to/asset]]
```

### Embed Code Files

Embed a code file with automatic syntax highlighting:

```markdown
[[code, ./assets/code/example.py]]
```

The file extension determines the syntax highlighting language.

### Embed Images

Embed an image:

```markdown
[[img, ./assets/images/architecture.png]]
```

### Embed Videos

Embed a video with controls:

```markdown
[[video, ./assets/videos/demo.mp4]]
```

## Path Resolution

Paths are **relative to the MDX file** containing the embed:

```
docs/01_guides/
├── 01_tutorial.mdx          ← From here
└── assets/
    └── code/
        └── example.py       ← Reference as ./assets/code/example.py
```

In `01_tutorial.mdx`:

```markdown
[[code, ./assets/code/example.py]]
```

## Complete Example

Given this structure:

```
docs/01_api/
├── settings.json
├── 01_authentication.mdx
└── assets/
    ├── code/
    │   ├── auth-example.py
    │   └── config.json
    └── images/
        └── auth-flow.png
```

In `01_authentication.mdx`:

```markdown
---
title: Authentication
description: API authentication guide
---

# Authentication

## Overview

Here's how the authentication flow works:

[[img, ./assets/images/auth-flow.png]]

## Code Example

Here's a Python example:

[[code, ./assets/code/auth-example.py]]

And the configuration:

[[code, ./assets/code/config.json]]
```

## Best Practices

1. **Organize by type** - Use subfolders like `code/`, `images/`, `videos/`
2. **Use descriptive names** - `auth-flow.png` not `diagram1.png`
3. **Keep assets close** - Store assets near the docs that use them
4. **Version control friendly** - Small, text-based files when possible

## Fallback: Manual Imports

If you need more control, you can still use manual imports:

```mdx
import ExampleCode from '!!raw-loader!./assets/code/example.py';

<pre><code>{ExampleCode}</code></pre>
```

But the `[[type, path]]` syntax is recommended for simplicity.
